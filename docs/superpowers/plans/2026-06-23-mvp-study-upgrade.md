# MVP Study Upgrade Implementation Plan

Goal: transform the app from queue-only progression into a practical study MVP with a vocabulary inventory screen and explicit sentence-repetition practice.

Architecture: keep the single-page app and IndexedDB persistence. Add explicit UI modes in `src/app.js`, keep scheduling and inventory selection in pure functions in `src/scheduler.js`, and verify behavior with Node tests.

Tech stack: vanilla JavaScript, shared CSS, Node test runner, GitHub Pages static deployment.

---

## Completed Tasks

1. Add deterministic inventory helpers

Files: `src/scheduler.js`, `tests/scheduler.study.test.js`

Implemented `getInventoryItems` and `summarizeInventoryCounts` so UI code can filter items by level, type, and state without embedding scheduling logic in DOM rendering.

Verification: `npm test tests/scheduler.study.test.js`

2. Add vocabulary inventory mode

Files: `src/app.js`, `src/styles.css`, `tests/static.test.js`

Added `vocabulary` UI mode with level, type, and state filters. The screen shows current state counts and lets the learner mark each item known, uncertain, or unknown.

Verification: `npm test tests/static.test.js`

3. Add sentence repetition drill mode

Files: `src/app.js`, `src/styles.css`, `tests/static.test.js`

Added `drill` UI mode. A learner can open practice from the queue or vocabulary list, repeat the linked Russian sentence aloud, replay speech synthesis when available, and mark recall outcome. Drill answers reuse `applyScreeningAnswer` and return to the session screen.

Verification: `npm test tests/static.test.js`

4. Document study-ready flow

Files: `README.md`

Updated README to describe the implemented vocabulary screen and sentence drill flow instead of describing sentence repetition as a future step.

Verification: `npm test`

5. Prepare deployment

Files: static output generated by `npm run web:prepare`

Run `npm test` and `npm run web:prepare`, then push to `main` so the existing GitHub Pages workflow deploys the app.

Verification: deployed URL returns an HTML page containing the Langtu app root and module script.
